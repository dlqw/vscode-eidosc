# 0.3.2-alpha.2

## Fixed

- Fixed TextMate highlighting for generic name-first declarations such as `Direction[A] :: type` and `step_x[A] :: Direction[A] -> Int`.
- Fixed extension-side qualified word ranges for dot-separated module paths such as `Pkg::Foo.Bar::item`.
- Fixed lexical semantic fallback module-prefix highlighting for dot-separated module paths.
- Fixed TextMate highlighting for multiline record and record-pattern field labels.
- Fixed stale semantic diagnostics being applied after the document changed during analysis.
- Fixed semantic refresh latency by preferring native/bundled `eidosc`, cancelling stale subprocesses, and using last-good snapshots for interactive providers.
- Added an opt-in long-lived LSP backend for diagnostics, completion, hover, definitions, references, document symbols, semantic tokens, code actions, and inlay hints.
- Avoided sending redundant LSP `didChange` notifications from repeated provider requests when the document version has not changed.
- Switched the default semantic backend to LSP with an IDE snapshot fallback cooldown when the LSP server cannot start or crashes.

## Changed

- Updated Eidos keyword, attribute, operator, and builtin type highlighting for the current 2026Q3.1 surface, including `effect`, `@derive`, `@ffi`, `@borrow`, `@operator`, `Seq`, and C ABI types.
